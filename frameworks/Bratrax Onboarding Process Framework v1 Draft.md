## **🧭 Bratrax Onboarding Plan (Dummy-First Method)**

### **1\. Pre-Flight (Days \-3 → 0\)**

**Goal:** Understand business & stack — no data access yet.  
 Ask client:

* What tools do you use? (Shopify, Klaviyo, ad platforms, etc.)

* What reports/metrics do you check daily or weekly?

* Who needs this dashboard and for what decisions?  
   Collect: screenshots of reports, sample spreadsheets, creative sheet.

---

### **2\. Discovery Call (Day 1\)**

**Goal:** Define what dashboard *should show*, not how it connects.

* Walk through their decision flow (budget, creative, performance)

* Identify 3–5 key metrics (nROAS, CAC, AOV, etc.)

* Map dashboard questions:  
   “What do you want to see first thing every morning?”  
   “What decision will this help you make?”  
   Outcome → draft list of **views \+ metrics.**

---

### **3\. Dummy Dashboard Build (Days 2–6)**

**Goal:** Prototype full experience using **mock data.**  
 Steps:

* Use pre-generated dummy dataset shaped like their tech stack

* Create **3 dashboards:**

  1. *Attribution Command Center*

  2. *Creative Leaderboard*

  3. *Journey Explorer*

* Simulate realistic values & patterns (spend, conversions, ROAS)

* Use dummy data to visualize layout, flow, filters, interactions  
   **Deliverable:** Clickable prototype that *feels real* but no pipelines yet.

---

### **4\. Review & Align (Day 7\)**

**Goal:** Validate what matters before touching data.

* Client walkthrough: “If this were live, would it answer your questions?”

* Gather exact feedback:

  * Which numbers are critical?

  * What’s missing?

  * What’s unnecessary?

* Lock final structure \+ metric definitions.

---

### **5\. Pipeline Build (Days 8–14)**

**Goal:** Connect only approved data flows.

* Build ETL / API pipelines *for confirmed dashboards only*

* Validate data formats & accuracy

* Replace dummy data with live feeds

* Re-test visualizations with real numbers

---

### **6\. Go-Live & Train (Days 15–18)**

* Daily refresh schedule

* Quick user guide: “How to read & act on each view”

* Verify metric parity (±5 %)

* Launch email \+ feedback loop

---

### **✅ Success Signals**

* Client approves design before any heavy data work

* Pipelines only built for dashboards that matter

* Feedback cycle \<7 days

* 2× faster delivery and zero wasted pipelines

---

### **🧠 Core Principle**

**“Design first, wire later.”**  
 Same as architecture: blueprint → mock-up → construction.  
 Dummy data removes engineering bottleneck and clarifies *what they actually want* before writing a single pipeline.
